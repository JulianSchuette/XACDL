What is it?

XACDL is a textual DSL for writing XACML policies and requests. It is developed using Xtext.

Why?

I had to write a lot of fairly simple XACML policies, and even more XACML requests. Doing it "manually" is boring and error-prone. I didn't like the GUI or web-based tools I could find either, because of their limitations and/or the need to use a mouse all the time. So I created my own DSL.

Limitations?

XACDL doesn't support the whole XACML standard, far from it. I'm adding new features as I need them, but contributions are of course very welcome.
